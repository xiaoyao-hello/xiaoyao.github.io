什么是babel

Babel 是一个 JavaScript 编译器工具，主要用于将当前（或未来）版本的 JavaScript 代码转换为向后兼容的旧版本 JavaScript 代码，以确保代码在不同浏览器和环境中能够正确运行。Babel 支持最新的 ECMAScript 标准规范，可以帮助开发者在项目中使用最新的 JavaScript 特性。Babel 还可以用于转换 JSX 代码、TypeScript 代码以及其他类 JavaScript 语言，使开发者能够更灵活地使用各种语言特性来开发应用程序。

babel的安装与使用

安装babel

babel可以和构建工具联合使用，也可以独立使用
如果要独立的使用babel，需要安装下面两个库：

  • @babel/core：babel核心库，提供了编译所需的所有api
  • @babel/cli：提供一个命令行工具，调用核心库的api完成编译

babel的使用

babel的配置

babel本身没有做任何事情，真正的编译要依托于babel插件和babel预设来完成
通过一个配置文件.babelrc

babel预设

babel有多种预设，最常见的预设是@babel/preset-env
@babel/preset-env可以让你使用最新的JS语法，而无需针对每种语法转换设置具体的插件

babel插件

除了预设可以转换代码之外，插件也可以转换代码，它们的顺序是：

  • 插件在 Presets 前运行。
  • 插件顺序从前往后排列。
  • Preset 顺序是颠倒的（从后往前）。

通常情况下，@babel/preset-env只转换那些已经形成正式标准的语法，对于某些处于早期阶段、还没有确定的语法不做转换。
如果要转换这些语法，就要单独使用插件
下面随便列举一些插件

@babel/plugin-proposal-class-properties

该插件可以让你在类中书写初始化字段

@babel/plugin-proposal-function-bind

该插件可以让你轻松的为某个方法绑定this

遗憾的是，目前vscode无法识别该语法，会在代码中报错，虽然并不会有什么实际性的危害，但是影响观

@babel/plugin-proposal-optional-chaining

babel-plugin-transform-remove-console

该插件会移除源码中的控制台输出语句

编译后

@babel/plugin-transform-runtime

用于提供一些公共的API，这些API会帮助代码转换
